ros_mqtt_bridge — README

What this repo contains
- A small ROS2 (Humble) Python package ros_mqtt_bridge that subscribes to GPS and system topics and publishes serialized protobuf Boat messages to an MQTT broker.
- Two mock ROS nodes: mock_gps and mock_system that publish the topics the bridge subscribes to.

Quick local run (native Linux / Linux VM)
1. Start an MQTT broker (e.g., mosquitto on localhost).
2. Install Python deps (if not using Docker):
   pip3 install -e .[install]  # from the package root or pip3 install paho-mqtt python-dotenv
3. Source ROS 2 and the workspace (after building with colcon):
   source /opt/ros/humble/setup.bash
   source install/setup.bash
4. Start mock nodes (each in its own terminal):
   ros2 run mock_gps mock_gps
   ros2 run mock_system mock_system
5. Start the bridge:
   ros2 run ros_mqtt_bridge ros_mqtt_bridge
6. Inspect MQTT messages:
   mosquitto_sub -h localhost -t "marinor/#" -v

Why Docker is recommended on macOS (Apple Silicon)
- ROS2 Humble does not ship native macOS/arm64 binaries. Running ROS2 inside Linux containers is the practical approach on Apple Silicon. Docker Desktop supports multi-arch images using buildx; prefer building for linux/arm64 when possible for best performance.

Docker: build and run (Apple Silicon recommended)
1. Build the image (prefer native arm64):
   docker build --platform linux/arm64 -t marinor_ros_ws .

2. Run the container interactively and mount the repo so you can iterate:
   docker run --rm -it \
     -v "$(pwd)":/root/ws \
     -e MQTT_HOST=host.docker.internal \
     marinor_ros_ws

3. Inside the container (in each shell):
   source /opt/ros/humble/setup.bash
   source /root/ws/install/setup.bash

   # run mock nodes (in separate shells)
   ros2 run mock_gps mock_gps
   ros2 run mock_system mock_system

   # run the bridge
   ros2 run ros_mqtt_bridge ros_mqtt_bridge

Notes for macOS networking
- To reach a broker on the macOS host, set MQTT_HOST=host.docker.internal when starting the container. This is supported by Docker Desktop.
- Alternatively run mosquitto in Docker and attach both containers to the same Docker network:
  docker network create ros-net
  docker run -d --network ros-net --name mosquitto -p 1883:1883 eclipse-mosquitto
  docker run --rm -it --network ros-net -e MQTT_HOST=mosquitto marinor_ros_ws

Troubleshooting tips
- 'Package "..." not found' after container start: the workspace install/setup.bash wasn't generated by colcon. Rebuild inside the container and inspect colcon output:
  colcon build --packages-select ros_mqtt_bridge mock_gps mock_system --event-handlers console_cohesion+
- If colcon fails with errors about environment hooks or missing template files: the repo previously referenced ament env-hook templates in setup.py; those references have been removed. Ensure your local setup.py changes match the repository.
- If MQTT messages don't arrive: ensure MQTT_HOST is set correctly (localhost vs host.docker.internal vs broker container name) and broker is reachable.
- To inspect installed packages inside container:
  source /opt/ros/humble/setup.bash
  source /root/ws/install/setup.bash
  ros2 pkg list | grep ros_mqtt_bridge

Advanced notes
- The bridge serializes a protobuf message (boat_pb2.py). Ensure protobuf support is present if you modify serialization logic.
- For development, prefer running colcon build locally in a Linux VM or inside the container; iterate by mounting the repo into the container and rebuilding.

If anything in these instructions doesn't work on your machine, paste the exact build or runtime logs and the commands you ran and troubleshooting steps will be provided.
